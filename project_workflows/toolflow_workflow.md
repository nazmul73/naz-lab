# ToolFlow End-to-End Workflow

ToolFlow is an AI tools, SaaS, automation, and productivity systems content project. It can remain English-first as a project preset, while Naz Lab as a whole remains Bangla-first.

## Global language behavior

- Naz Lab default: Bangla-first
- ToolFlow preset: English-first by default
- Bangla version: supported whenever requested
- Other languages: optional

## Brand voice

ToolFlow should feel:

- modern
- clean
- practical
- premium
- trustworthy
- useful
- concise
- non-hype

Avoid:

- fake income claims
- spammy marketer tone
- unverified AI news
- crypto speculation
- political content
- random gadget reviews
- heavy coding unless beginner-friendly
- generic motivation fluff

## Workflow chain

1. Tool/topic intake
2. Content angle
3. Main Facebook post
4. Carousel version
5. Reel script
6. Image/cover prompt
7. Voice package
8. Video package
9. Caption/CTA/hashtags
10. Final posting package

## Step 1 - Tool/topic intake

Input can be:

- AI tool name
- SaaS product
- automation idea
- productivity workflow
- comparison topic
- review topic

Required fields:

- tool/topic
- target audience
- use case
- platform
- language target
- content format

## Step 2 - Content angle

Recommended angles:

- beginner-friendly workflow
- save time with a system
- tool comparison
- practical review
- before/after productivity flow
- common mistake and fix
- simple automation tutorial

## Step 3 - Main Facebook post

Structure:

- hook
- problem
- practical solution
- step-by-step value
- quick verdict
- CTA question

Tone:

- clear
- useful
- non-hype
- practical
- trustworthy

## Step 4 - Carousel version

Recommended slide structure:

1. Hook / promise
2. Problem
3. Tool or workflow
4. Step 1
5. Step 2
6. Step 3
7. Result / benefit
8. CTA

## Step 5 - Reel script

Recommended structure:

- 0-3s hook
- 3-10s problem
- 10-25s workflow steps
- 25-30s result / CTA

## Step 6 - Image/cover prompt

Visual style:

- clean modern SaaS/productivity aesthetic
- dashboard/workspace context
- premium minimal composition
- readable but not text-heavy
- no fake logos unless provided
- no misleading product UI

## Step 7 - Voice package

Voice direction:

- confident but not salesy
- practical explainer voice
- clear pacing
- clean spoken English by default
- Bangla version should be natural and simple if requested

Reference voice:

- only user-provided or explicitly authorized reference audio

## Step 8 - Video package

Video structure:

- Scene 1: messy workflow/problem
- Scene 2: tool/system introduction
- Scene 3: 2-3 steps
- Scene 4: result/benefit
- Scene 5: CTA

Edit style:

- creator explainer
- screen/dashboard friendly
- clean cuts
- readable subtitles
- minimal effects

## Step 9 - Caption/CTA/hashtags

CTA examples:

- Would you use this workflow?
- Which step would save you the most time?
- Want a simple version of this setup?

Hashtag examples:

- #AITools
- #Productivity
- #Automation
- #SaaS
- #Workflow
- #NoCode
- #DigitalTools
- #ToolFlow

## Final posting package fields

- project_preset
- language
- topic/tool
- audience
- use_case
- main_post
- carousel
- reel_script
- image_prompt
- voice_direction
- video_direction
- captions
- CTA
- hashtags
- output_paths
