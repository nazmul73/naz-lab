# Naz Lab Bangla Quality Engine

Naz Lab is Bangla-first by default. This guide defines the Bangla quality rules that should be used across Text, Image planning, Voice, Video, Portrait, and project workflows.

## Priority

1. Bangla first
2. English second
3. Other languages optional

## Core Bangla rule

Bangla output must sound natural, spoken, useful, and ready to publish.

Avoid stiff textbook Bangla unless the user explicitly asks for formal writing.

## Required Bangla qualities

Bangla should be:

- natural
- simple
- spoken
- Facebook-ready
- netizen-friendly
- voiceover-ready
- emotionally clear
- not robotic
- not over-formal
- not overly literary unless requested

## Default regional priority

Primary regional flavor:

- Rangpur / Nilphamari / North Bengal

Secondary regional tones:

- Dhakaiya
- Chattogram
- Sylhet
- Noakhali / Comilla

Regional tone should be light and natural, not exaggerated.

## Natural spoken Bangla

Use short, clear sentences.

Prefer:

- তুমি
- আমরা
- এই ব্যাপারটা
- আসলে
- খেয়াল করলে দেখবে
- তোমার কী মনে হয়?

Avoid unnecessary heavy words:

- অতএব
- পরিশেষে
- নিম্নোক্ত
- উক্ত বিষয়
- aforementioned-style Bangla

## Facebook / netizen Bangla

Use conversational, social-friendly phrasing.

Good patterns:

- এই জায়গাটাই সবচেয়ে গুরুত্বপূর্ণ।
- মানুষ আসলে এখানেই ভুল করে।
- ব্যাপারটা বাইরে থেকে সহজ মনে হলেও ভিতরে অন্য গল্প আছে।
- তুমি হলে কী করতে?
- তোমার চোখে কোন detailটা আগে পড়ত?

Avoid:

- spammy hype
- fake urgency
- clickbait that misrepresents facts
- overly formal newsreader tone

## Voiceover-ready Bangla

Voiceover Bangla must be easy to say out loud.

Rules:

- one idea per sentence
- short clauses
- clear pause points
- no long tangled sentence
- avoid repeated filler
- avoid tongue-twisting phrasing

Example structure:

Hook:
এই কথাটা অনেকেই খেয়াল করে না।

Body:
ঘটনাটা বাইরে থেকে সাধারণ মনে হলেও, ভেতরে একটা বড় প্রশ্ন লুকিয়ে ছিল।

CTA:
তোমার কী মনে হয়?

## Rangpur / Nilphamari / North Bengal tone

Use lightly. Do not overdo dialect.

Acceptable flavor:

- একটু নরম, সহজ, উত্তরবঙ্গের কথার ভাব
- direct but warm
- less polished Dhaka media tone
- grounded and human

Do not force heavy dialect spellings unless user explicitly asks.

Sample flavor:

- ব্যাপারটা কিন্তু একদম সহজ না।
- এই জায়গাতেই মানুষ ভুল বোঝে।
- তুমি হলে আগে কোনটা ধরতে?

## Dhakaiya tone

Use only when requested.

Style:

- faster, urban, casual
- social media friendly
- not caricatured

## Chattogram tone

Use only when requested.

Style:

- local flavor without exaggeration
- natural phrase rhythm
- avoid making it comic unless requested

## Sylhet tone

Use only when requested.

Style:

- light Sylheti influence
- maintain readability for broad Bangla audience

## Noakhali / Comilla tone

Use only when requested.

Style:

- conversational regional flavor
- avoid over-stereotyping

## True Noir Tales Bangla

When True Noir Tales uses Bangla:

- keep it suspenseful but restrained
- simple and emotional
- no gore details
- no dead body description
- no visible wound language
- end with a question when useful

Example:

ঘটনাটা বাইরে থেকে খুব সাধারণ মনে হয়েছিল। কিন্তু একটা ছোট detail পুরো গল্পটার অর্থ বদলে দেয়। তুমি হলে প্রথমে কোন জিনিসটা খেয়াল করতে?

## ToolFlow Bangla

When ToolFlow uses Bangla:

- practical
- clean
- useful
- non-hype
- beginner-friendly
- no fake income claims

Example:

AI tool আলাদা আলাদা ব্যবহার করলে ফল random হয়। কিন্তু একটা simple workflow বানালে output অনেক পরিষ্কার হয়। তুমি কি এই setupটা ব্যবহার করবে?

## Prompt rule for future apps

Every Bangla generation workflow should include this instruction:

"বাংলা হবে natural spoken Bangla, Facebook-ready, netizen-friendly, voiceover-ready, simple, human, and not stiff textbook Bangla. Regional tone needed হলে default হবে Rangpur/Nilphamari/North Bengal flavor, lightly and naturally."
